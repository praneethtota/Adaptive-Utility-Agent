The Framework

Utility as a control law, not a monitor

The utility function U = w_e(f)·E + w_c(f)·C + w_k(f)·K governs everything: how strongly each detected error is penalized at training time, when behavioral corrections are injected into the system prompt, whether a new model deployment passes acceptance, and whether the system should act or abstain. It is the same formula across every domain — only the field weights change.

U = w_e(f) · E + w_c(f) · C + w_k(f) · K E — Efficacy: performance relative to a human baseline [0, 1] C — Confidence: internal consistency, penalized by contradictions [0, 1] K — Curiosity: exploration bonus for novel high-upside domains [capped at 50%] f — field: surgery / software / creative / AV / energy / ... Decision rule: act if C ≥ C_min(f) AND E ≥ E_min(f) abstain otherwise → escalate to human or safe state Field-specific penalty multipliers route into DPO training: Surgery contradiction → 10× training weight Software engineering → 2× training weight Creative writing → 1× training weight

The additive weighted structure is not a design choice — it is the unique functional form required by five behavioral axioms, proved from first principles via Debreu's representation theorem (Theorem B.1, Appendix B). Field weights are derived from professional licensing standards, not arbitrary parameters. See §4–5 of the full whitepaper.

Layer 1 · Milliseconds

Session correction

Contradiction detected → corrective assertion stored → injected into system prompt. Immediate, no weight change. Effect is session-scoped but persists across sessions via the assertions store.

Layer 2 · Hours

Calibration cycle

Utility-scored pairs → field-penalty-weighted DPO fine-tuning → LoRA adapter → benchmark gate → deploy. Weight-level, permanent, generalizing. A surgical error trained 10× harder than a creative writing mistake.

Layer 3 · Monthly

Release integration

Accumulated adapters distilled into new base fine-tune. Full regression suite. New starting point for the next correction cycle. Wrapper-level learning baked into base model weights.

Quick Start

No API key, no GPU, no special setup

The simulation runs on standard scientific Python. The full pipeline — utility scoring, contradiction detection, assertions store, DPO pair collection, personality evolution — runs today on any laptop.

Step 1 · Install

Clone and install

Standard scientific Python stack. No GPU required for any simulation.

git clone https://github.com/praneethtota/\
  Adaptive-Utility-Agent.git
cd Adaptive-Utility-Agent/agent
pip install numpy scipy matplotlib
Step 2 · Simulate

Run the correction loop

3-cycle simulation: seeded O(n²)→O(n) contradiction detected in cycle 1, resolved in cycle 2, +0.1160 utility improvement by cycle 3.

# 3-cycle simulation
python3 simulate.py

# Extended 500-task two-arm study
python3 simulate_extended.py
Step 3 · Route

Run the routing experiment

Four-arm study: oracle +12.5%, VCG +10.5%, mismatched −17.5%. Outputs JSON + 4 figures.

python3 routing_experiment.py
# → routing_output/routing_results.json
# → routing_output/plots/ (4 PNG figures)

Live harness (requires API key): export ANTHROPIC_API_KEY=sk-ant-... && python3 harness.py — runs LeetCode-style problems through the full pipeline with real Claude responses, automated scoring, and DPO pair export.
